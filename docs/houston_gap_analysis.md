## 可能原因与对应措施

### 1. 数据划分（影响最大）

**论文**：toy 实验使用 **95% 训练 / 5% 验证** 随机划分。

**当前**：Houston 固定划分（2832 训练 / 12197 测试），训练样本远少于 95% 划分。

**建议**：改用 95/5 随机划分，增加训练样本：
```bash
python sample_dataset_houston_full.py --split_mode random_95_5
```

### 2. 联邦学习设置

**论文**：16 个 client（8 HSI + 8 LiDAR），梯度在 client 间聚合。

**当前**：单进程（WORLD_SIZE=1），等价于 local-fusion，无联邦梯度交换。

**说明**：复现 16 client 需多进程/多机，实现成本高。单 client 通常略逊于联邦设置。

### 3. 数据来源差异

**论文**：可能使用官方 Houston 2013 竞赛数据。

**当前**：使用 rs-fusion-datasets 下载的数据，预处理与官方可能不同。

**建议**：若条件允许，可对比官方数据与 rs-fusion-datasets 的差异。

### 4. 随机性与多次实验

**论文**：可能报告多次运行的平均或最优结果。

**建议**：用不同 seed 多次训练，取平均或最优：
```bash
# 例如 seed 42, 123, 456 各跑一次
```

### 5. 超参数微调

可尝试：
- `--with_svd 0`：对比 Non-SVD 与 SVD 版本
- `--svd_k 1` 或 `--svd_k 3`：论文中 K=2 为推荐值，可做消融
- 学习率：0.0005 或 0.002
- 训练轮数：400 或 500
